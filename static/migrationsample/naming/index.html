New Naming Conventions

The table shows the abbreviations for all new pdfs and images on AAP.org submitted through AAP.org/websites or brought over as part of a site migration. If you are a content owner for an AAP.org subsite and do not see your name, please contact editorial staff to be added. Each column can be sorted by clicking .

"Abbreviation" is the most important column. You will use this as the prefix for PDFs and image filenames. Filenames for web pages should be short and relevant to the content.

  • Filenames are lowercase
  • PDFs and images for a site begin with the same abbreviation
  • Meaningful spaces in filenames will be marked with an underscore (ie, no hyphens or white spaces)
  • Delete version numbers or Reviewer Initials
  • Dates should be avoided as well, with the exception of periodicals (eg, notes from a yearly meeting, newsletters, annual fact sheets (in some cases), etc.)

Note: Recurring content (page names for new sites, newsletters, fact sheets, headshots, etc) will have more extensive naming conventions.

PDFs

PDFs should take the form of the naming convention abbreviation followed by an underscore followed by a short, consistent description:

  • federaladvocacy_chip_principles.pdf
  • federaladvocacy_chip_templatelettertogovernors.pdf
  • 9.12.14combinedfederalupdate.pdf

Moving forward, all PDFs are going to be stored at the root. This means that if you give you name your PDF according to the new naming convention, you will have a permanent link for that document to use in the communication medium of your choice. Just prepend "http://www.aap.org/en-us/Documents/" to your PDF and you're set. Continuing with the above examples:

  • http://www.aap.org/en-us/Documents/federaladvocacy_chip_principles.pdf
  • http://www.aap.org/en-us/Documents/federaladvocacy_chip_templatelettertogovernors.pdf

Submit your PDFs via the site requests at AAP.org/websites or find out how to submit new content.

Newsletters

Some content types are common enough across the site collection that they merit more extensive naming conventions.

Newsletter filenames begin with the site abbreviation from above. However, unlike most PDFs, newsletters can—and should—include a chronological reference. The format is either volume-issue (vv_ii) or just volume/season/month, depending on frequency of publication:

  • abbreviation_newsletter_yyyy_vv_ii
  • socc_newsletter_2014_04_01
  • sohm_newsletter_2012_summer
  • newsletter_FINAL
  • Newsletter_SOHM_july7-07-14
  • sohm_7-4-14_newsletter

Fact Sheets

The same rule applies for fact sheets (another common content type in the CMS), with the addition of spelling out the full state name:

  • abbreviation_factsheet_location
  • federaladvocacy_medicaidfactsheet_illinois
  • federaladvocacy_medicaidfactsheet_IL
  • medicaidfactsheet_IL
  • medicaidfactsheet_illinois

That said, yearly fact sheets might not merit including a year in the filename. For example, if an outside user wants to connect to the Illinois Medicaid Fact Sheet on AAP.org, creating a more consistent URL is better for future-proofing. E.g.—

  • aap.org/en-us/Documents/federaladvocacy_medicaidfactsheet_illinois

Can easily be replaced year to year. Whereas

  • aap.org/en-us/Documents/federaladvocacy_medicaidfactsheet_2014_illinois

Will eventually result in a dead link. Although some content is archivable on AAP.org, filenames (for both PDfs and web pages) should be named such that being directed to them will result in the user getting the most recent and relevant information. Remember that everything on the site is crawled by search.

NCE

The month before NCE is one of the busiest times of year for Editorial. We eceived a lot of requests for uploading documents, and images. For this reason, we ask that you add "nce_" to the naming convention.

  • socc_nce_agenda.pdf
  • sohm_nce_membershipinformation.pdf
  • NCE Agena Book FINAL.pdf
  • Section meeting group photo.jpg

Web Pages

For those adding new pages to AAP.org or currently working on migrations, there are a few simple rules to creating names for your web page:

  1. Shorter is better

  2. Create the type of intuitive names you'd expect to see in a vanity URL

  3. home.docx

  4. about.docx
  5. news.docx
  6. helping_babies_breathe.docx
  7. about_helping_babies_breathe.docx
  8. news_from_our_members.docx

If you are creating multiple web pages in the form of MS Word documents and want the pages to link to one another—a very good thing—just use the short page name followed by ".aspx" in your link, and create a hyperlink the way you normally would in Microsoft Word.

Images

Images follow the same format as PDFs and require front matter when submitted to Editorial:

  • commpeds_randomimage.png
  • commpeds_groupphoto2014retreating_final_v7.jpg

There are a couple exceptions that are very important: headshots of members and logos. Headshots are formatted as headshot_lastname_firstinitial_middleinitial. Logos are prefaced with _logo*:

  • headshot_watters_r_d.png
  • logo_prep.png
  • Ryan-Watters.jpg
  • QuIInlogo FINAL.jpg

Note: If you're submitting a new logo to AAP.org, please email the logo directly to Ryan Watters. Make sure that that logo is in the original design file type (ie, .eps, .svg) so that Editorial can resize or edit the logo as needed without losing quality. Look to the New Image Standards for more information.
